How ASP.NET works Sample Configuration

To configure some of these examples you should do the following:

  • Create Virtual Directory
    Create a virtual for this directory in the IIS Management Console to get it ready to be accessible by ASP.Net.

  • Directory Security - full access for Web account
    Several of the examples require WRITE access in the Web directory of this application. Both the Config File class and the Logging Class require this to work properly. To configure, please allow READ/WRITE/CREATE access to the ASPNET/NETWORK SERVICE account in this directory.

  • SQL Connection String for SQL Logging
    The SQL Logging routine requires a SQL Connection string which is stored in the Config file via the wwAppConfiguration class. Please set the ConnectionString property of the App class in the Global.asax.cs file to a valid database/account that allows creation of the logging table - you will be asked to create the table from the Log Display form.

  • Http Handler Configuration
    Several of the examples require that the HTTP Handlers and Modules are enabled. The handlers are pre-configured and commented out in the web.config file, so you can simply un-comment them as needed. There are SharewareMessage and ImageSample Modules . The Url Rewriting handler is used for the .wws to .aspx mapping example. Remember to also set the .wws extension to the aspnet_isapi.dll in the IIS Mgt. console.

  • Image Watermark Sample You need to configure .jpg to be mapped as a script map in IIS and mapped to aspnet_isapi.dll (like .aspx). Then make sure the ImageModule module is enabled in web.config.
